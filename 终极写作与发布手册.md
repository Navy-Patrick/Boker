# 终极写作与发布手册（唯一维护文档）

适用目标：你只专注写作，索引、目录、页面展示、日常发布自动完成。

---

## 0. 一句话原则

- 只写 Markdown
- 文件名决定标题和日期（索引直接扫描文件名）
- 每天最多一条命令完成更新与发布

---

## 1. 目录约定（不要乱改）

```text

├─ content/

│  └─ poems.js                        # 手动维护诗句库
├─ scripts/

├─ style.css / app.js / package.json
└─ 终极写作与发布手册.md
```

## 2. 文件命名标准（索引核心）

## 2.1 随笔（必须）

放在 `content/essays/`，命名：

```text
YYYY-MM-DD-标题.md
```

例如：

```text
2026-04-13-今天把节奏慢下来.md
```

索引提取规则：
- 日期：取文件名里的 `YYYY-MM-DD`
- 标题：取日期后面的文字

## 2.2 读书（建议）

放在 `content/reading/daodejing/...`：

- 可直接用文件名：`YYYY-MM-DD-道德经--第十九章.md`
- 也支持爬虫目录：`[2026-04-13] 道德经--第十九章 (...)/index.md`

索引提取规则：
- 日期：优先从目录名/文件名中的 `YYYY-MM-DD`
- 标题：优先取目录名（若文件是 `index.md`），否则取文件名
- 章节：从标题里的“第X章”自动识别

## 3. 写作模板（正文可自由）

## 3.1 随笔模板

```md
# 标题（可写可不写，页面顶部标题来自文件名）

今天正文第一段。

## 小标题

- 要点 1
- 要点 2
```

## 3.2 读书模板

```md
# 标题（可写可不写）

**原文**：

……

**译文**：

……

**个人体会**：

……
```

说明：
- 页面会自动隐藏作者/来源/日期元信息行
- `**原文**` / `**译文**` 会正常加粗

## 4. 日常命令（最常用）

## 4.1 仅更新索引

```bash
npm run build:indexes
```

## 4.2 本地发布（更新索引 + add + commit）

```bash
npm run publish
```

## 4.3 直接推送 GitHub（更新索引 + add + commit + push）

```bash
npm run publish:push
```

---

## 5. 首次接入 GitHub（只做一次）

如果终端提示“not a git repository”，先执行：

```bash
git init
git add .
git commit -m "init: blog"
git branch -M main
git remote add origin 你的仓库地址
git push -u origin main
```

后续每天只要 `npm run publish:push`。

## 6. 首页诗句库维护

文件：`data/poems.js`

格式：

```js
window.__POEMS__ = [
  { text: "莫听穿林打叶声，何妨吟啸且徐行。", author: "苏轼《定风波》" },
  { text: "且将新火试新茶，诗酒趁年华。", author: "苏轼《望江南》" }
];
```

每次进入首页随机展示。

## 7. 页面显示规则（当前已实现）

- 文章页只显示一套标题和时间（顶部）
- 正文自动过滤：
  - `Author / 作者`
  - `Source / 来源`
  - `Date / 日期`
  - `--` / `---`
- 正文加粗、列表、引用正常渲染

---

## 8. 故障排查

## 8.1 页面没内容

先执行：

```bash
npm run build:indexes
```

再强制刷新浏览器（`Ctrl + F5`）。

## 8.2 发布脚本报 git 错

说明当前目录尚未 git 初始化。按第 5 节初始化一次即可。

## 8.3 标题/日期不对

优先检查文件名是否符合规范：

```text
YYYY-MM-DD-标题.md
```

```bash

```
